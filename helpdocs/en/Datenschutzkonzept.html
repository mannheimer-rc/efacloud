Data security concept

An application on the Internet needs to be protected from data protection. For this purpose, a data protection concept is necessary.

The data protection concept should have two parts: a technical part technical part, which states how the application as a whole is secured against against misuse. This information usually comes from the web hoster or are covered by an agreement on (AVV).

In addition, a data protection concept is needed.

In addition, it requires an application-specific declaration of data data security and the use of data. For this purpose, efaCloud can automatically generate a data protection concept. It shows the current the current configuration, the authorised users with strong roles and the methods of protection.
