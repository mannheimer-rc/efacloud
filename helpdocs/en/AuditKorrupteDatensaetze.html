Corrupt records, missing information

Data records are checked for completeness. If values are missing in data fields, the fields in which the values are missing are displayed.
Corrupt is a data record that is missing technically relevant information.
If empty, it is automatically deleted.
A corrupt record that is not corrupt record that is not empty should be examined and, as a rule, usually deleted afterwards anyway. If data set keys are missing, making synchronisation with an efa-PC impossible, the problem is shown in this missing data section. Such data records can be corrected under certain circumstances by making a new entry.
